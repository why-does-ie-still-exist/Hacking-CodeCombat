Step One: Finding Something Interesting

This is my understanding of how hacking a system goes:

Interesting things can be defined in many ways, but in software I find these things draw attention(and are more often easy-to-exploit points of vulnerability) than others

  • Some code is too Generic
  • Some code is too Reused
  • The code is close to or completely readable

When it comes to Code Combat’s Campaign mode(the thing I will hacking today), there are many areas of code that satisfy these conditions, but one particular one caught my eye. That bit of code was a button that was present on every unlocked button…

Step Two: Think Like a Software Developer

If you go into chrome developer tools(Right Click > Inspect Element), and inspect element on the button, you find that its code is as follows for every level you enter:

<button data-i18n="common.play" class="btn btn-success btn btn-lg btn-illustrated start-level">Play</button>

And if you look at the pop-ups for the levels you haven’t unlocked yet, they look almost exactly like their unlocked counter parts, except they’re missing a button. So what if the button is the missing piece to unlock the level?

To explain my thinking, if I were a software developer and I wrote a web application, I would have one “play” button that would trigger the opening of the currently selected level, and put that on all the unlocked levels. This would simplify the design immensely, as you would only need one button click event listener and on triggering it you simply look at which level is being accessed. However, if you pressed the button on an locked level, what would happen?

Step Three: Act Like a Software Developer (Testing Your Idea)

So if our hunch is correct, all we need to do is move the button onto an unlocked level. If you try and move the button to an unlocked level, it works! Hooray! (If you want to do this, copy the above button code, click on an unlocked level and Right Click > Inspect Element its popup. Then replace any of the text elements with the button, and you’re in business!)

Step Four: Proof of Concept

Now that you know that the concept works, write some code that automates it! I made a chrome extension, and you can use your newfound hacking skills to find bugs in Code Combat. Remember: if you find a bug, you should always report it to the owner of the website. That’s what good programmers do.
